# 变更记录

说明：

- 本仓库采用独立版本控制
- 上游版本仅用于标记当前主要对齐基线
- 本仓库的公开整理版、热修版、功能增强版按自己的版本号继续演进

## 2026-04-18

### 已完成

- 代理页重构为子标签结构，拆分为通用、Clash、v2rayN、v2rayA、HTTP 动态代理五个区域，减少不同代理模式混在一起的混乱感
- v2rayA 页面补充运行环境配置项：
  - 面板地址
  - API 登录名与密码
  - Xray 可执行文件路径
  - geodata 目录
  - 环境变量文件路径
- v2rayA 模式文案统一改为“自动切点 + 面板管理”，不再误导为纯手动模式
- v2rayA 节点区改为固定高度表格，支持：
  - 表头固定
  - 内部滚动
  - 分页浏览
  - 当前节点高亮
  - 一键切换
- v2rayA 页面新增双视图：
  - 全部节点
  - 订阅分组
- v2rayA 全部节点视图新增类似 v2rayN 的顶部分组快捷筛选，可快速切换到某个订阅下查看全部节点
- v2rayA 订阅分组视图新增分组总表，可直接查看：
  - 分组名称
  - 节点数
  - 当前节点数
  - 最佳延迟
- v2rayA 自动切点链路继续接入现有代理主流程，业务运行时在启用智能切点且 `client_type=v2raya` 时会走自动切换逻辑
- 修复 v2rayA 节点解析逻辑，将 `subscription -> servers` 的上下文关系正确传递到子节点
- 修复 v2rayA 页面只显示“未分组”或节点总数异常的问题，改为正确识别全部订阅组与全部节点
- 修复代理页底部内容被右下角“保存并热重载配置”悬浮按钮遮挡的问题
- v2rayA 页面新增节点整理功能：
  - 标记失效
  - 清空失效标记
  - 移除重复
- v2rayA 页面新增“标记所有无延迟节点失效”按钮，适合在刷新全部延迟后快速批量清理测速失败节点
- v2rayA 节点表格新增状态识别：
  - 当前节点
  - 候选节点
  - 失效标记
  - 重复节点
- v2rayA 的失效标记接入项目运行态，后续自动切点会真正跳过这些节点
- v2rayA 的重复节点处理改为按 `类型 + 地址 + 端口` 归并，并将重复项批量标记为失效
- 智能切点总控中的“启动前先筛选可用节点 / 筛选缓存分钟数 / 单次最多筛选节点数”回归到通用页，避免在 v2rayN 子页重复维护
- 修复 v2rayA 节点切换时 `unhashable type: 'dict'` 异常，避免预检和自动切点过程中切换动作直接失败
- 进一步明确 v2rayA 预筛选开关行为：
  - 开启时执行批量预检并构建活节点池
  - 关闭时直接从候选节点池随机抽选并仅验证当前抽中的节点
- 集群总控页面重构为“子控列表 + 当前详情”结构，不再把所有子控日志大面板直接铺满页面
- 集群列表新增：
  - 搜索节点名/模式
  - 只看在线
  - 在线数 / 总数 / 当前可见详情数统计
  - 全部展开 / 全部隐藏详情
- 集群详情区新增：
  - 上一个子控
  - 下一个子控
- 集群页面的搜索、只看在线和子控显隐状态改为浏览器本地持久化
- 移除 v2rayA 工具区中的“重新筛选节点”按钮，避免误触发整池预检导致页面卡顿或误解为常规日常操作
- 代理页样式继续收敛为更清爽的后台配置风格，减少 Clash / v2rayN / v2rayA 三个子页之间的视觉割裂
- 将 v2rayN 子页从 Clash 共用模板中拆出，只保留 v2rayN 自身需要的配置和按钮
- 移除 v2rayN 子页中与 Clash 强绑定的功能与文案：
  - 控制台 API 地址
  - 代理策略组名称
  - 鉴权 Secret
  - 独享池模式
  - 延迟优选
- 集群总控新增主控侧“断开连接 / 允许连接”能力，可直接控制子控是否继续允许接入
- 集群列表和详情区同时增加连接状态标识与断开/恢复按钮
- 修正集群页列表列宽与按钮排布，减轻详情区被截断和拥挤的问题
- 继续优化 v2rayA 页面栅格布局，减少左右两栏不均衡和空白过大问题
- 继续优化 v2rayA 页面布局，将环境摘要卡上移到左侧主列下方，消除中间大面积空白区域
- 继续优化 v2rayA 页面布局，将摘要状态卡改为整排铺开，进一步减少页面中部空白
- 继续调整 v2rayA 页摘要卡位置，将环境摘要进一步并入上方结构，减少左侧与中部断层感
- 继续调整 v2rayA 摘要卡位置，将 Xray 路径和默认代理移入右侧工具区下方以填充右侧空白
- 继续优化 v2rayA 工具区和摘要卡相对位置，使右侧工具区下方空白进一步减少
- 公共发布版本号更新为 `v10.1.20`
